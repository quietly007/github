# Phase 1E — Master AIDE repair: PASS (with observational deferral)

**Date:** 2026-04-23

## Root cause
AIDE daily integrity check on Master had been failing with exit code 7 for weeks.
Root cause: the baseline database `/var/lib/aide/aide.db` was from January 2026. After
months of package updates (128,155 change lines in the last report), AIDE correctly
flagged the drift — `exit=7` means "added + removed + changed files detected", which
systemd interprets as service failure.

Secondary cause: volatile runtime files (uptime-kuma sqlite, traefik access.log,
vscode-server state) mutate DURING the 76-minute scan, producing "hash could not
be calculated" warnings that will trigger exit=1 on every future run regardless of
baseline freshness.

## Remediation
1. **New baseline generated** via `aideinit`. Post-init `aide --check` returned
   exit code **0** (clean). New DB size: 122 MB (up from 25 MB — reflects the
   much larger /home/qui3tly/.docker tree that has grown since January).

2. **Exclusion drop-in added** at `/etc/aide/aide.conf.d/99_local_excludes`:
   - `/home/qui3tly/.docker/uptime-kuma/data/.*`  (live sqlite WAL files)
   - `/home/qui3tly/.docker/traefik/logs/.*`      (active HTTP logs)
   - `/home/qui3tly/.vscode-server/.*`            (IDE runtime state)
   - `/var/lib/docker/containers/.*`              (docker container state)
   - `/var/lib/docker/overlay2/.*`                (overlay fs churn)

   Config validates via `aide --config-check` exit 0.

3. **Second baseline re-init** started with exclusions in place (running in
   background as of 21:16 CEST, ~76 min). This produces a final DB excluding the
   noise sources.

## Observational deferral
The nightly `aide.timer` fires at **00:13 CEST** (next: 2026-04-24). If the
exclusion list is correct, that run should exit 0 and the service moves from
`failed` → `active (inactive/waited)`. Evidence to collect tomorrow:
- `systemctl status aide.service` showing "SUCCESS" exit
- `journalctl -u aide.service -n 1 --since 00:00` empty or only benign

## Rollback
- `sudo rm /etc/aide/aide.conf.d/99_local_excludes` to revert exclusions
- `sudo cp /var/lib/aide/aide.db.bak /var/lib/aide/aide.db` (if old db was backed up — in our case aideinit overwrote it; /var/lib/aide/aide.db.new was cp'd to aide.db at 17:01)
