## Note on scope of UFW vs Docker

Docker manages its own iptables rules (DOCKER-USER, DOCKER). Container-to-container
traffic is NOT gated by these UFW rules at all. UFW only gates:
- Traffic to the host's own network stack (host-level services + docker-proxy 0.0.0.0 bindings)
- Return path of the host's outbound connections

Therefore this hardening reduces surface for host-level services, not container-to-container.
Broader segmentation (DOCKER-USER chain) is a Phase 5 task.
